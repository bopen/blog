B-Open technologies and best practices

What technologies and best practices are common at B-Open in 2022

Alessandro Amici
Jun 14, 2022 2 min read

At B-Open we try to keep up with the software development best practices and tooling, usually drawing from what larger Open Source projects are using successfully. Sometimes we need to adapt to the technological choices of our customers (JIRA comes to mind), but in 2022 we have several new projects starting up and most of them share a common set of technologies and best practices.

The main differences with respect to the past year are:

  • Kubernetes more or less everywhere (but still a lot of docker-compose)
  • much more GitHub and GitHub Actions ðŸŽŠ
  • FastAPI has taken over most API development
  • Ansible has kicked Puppet out of the list of legacy tools we need to endure ðŸ¥³
  • much less ReST in favour of MarkDown, but always with Sphinx

And here are the technolgies for 2022.

Collaboration

  1. Software project management: GitHub, GitLab, JIRA
  2. Source control: Git
  3. Documentation: MarkDown, Sphinx / MyST

Python

  1. Programming languages: Python 3.7+
  2. Python installation tooling: Pip, Conda
  3. Software testing: Pytest (unit testing), coverage.py (code coverage), Black (coding style), Mypy (type check)
  4. Scientific stack: Numpy, Pandas, Jupyter notebooks, Xarray, Dask, Dask.distributed, Scipy, Matplotlib, Plotly
  5. Geospatial stack: GDAL / OGR, rasterio, QGIS
  6. Web API stack: FastAPI, SQLAlchemy, Keycloak / WSO2

System / DevOps

  1. Operating Systems: Ubuntu Linux, CentOS Linux, MacOS, Windows.
  2. Containers: Docker, Docker compose, Kubernetes
  3. Cloud platforms: OpenStack, Google Cloud Platform, Microsoft Azure, Amazon Web Services
  4. Deployment automation: Ansible, Terraform
  5. Continuous integration: GitHub Actions, GitLab CI/CD, pre-commit
  6. Database: PostgreSQL / PostGIS

JavaScript

  1. Framework: React
  2. Web mapping: Leaflet
  3. End-to-end testing: Cypress
